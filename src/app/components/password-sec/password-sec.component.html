# ITGC Name Reference Control Frequency Risk
1 Password and system security ITGC-8 Yearly Low
Test Details
# ITGC Name Risk Classification Objective EY Testing method
1 Password and system security Unauthorized access to Database/Data file High - I - Fraud - Passwords to databases and other critical security configurations are set in an effective manner and periodically compared to appropriate requirements. Inspection
Test attributes

Step 1

A- Check the existence and relevance of DB password security guidelines.

Step 2

B- Verify that the use of a password is mandatory for all "Key Applications" and infrastructures ones and the table of passwords is encrypted (observation) If encryption of the table is not possible, ensure that access to this table is restricted to appropriate staff.3. Password controls are regularly compared to appropriate requirements, at least once a year.

Step 3

C- For each relevant technical component of the logical access path, obtain evidence of the organization’s settings for the following security configurations: • Minimum password length • Initial log-on uses a one-time password • Password composition (e.g., alpha/numeric characters, not words in dictionary) • Frequency of forced password changes • The number of unsuccessful log on attempts allowed before lockout • Ability of users to assign their own passwords • Number of passwords that must be used prior to using a password again • Idle session time out • Logging of unsuccessful login attempts
