Test Details
# ITGC Name Risk Classification Objective EY Testing method
1 Change Management Process Unauthorized changes to application programs High - I - Fraud - Change requests are documented and must be authorized and adequately tested before being applied to production Reperformance
Test attributes

Step 1

A formal, up-to-date and documented procedure for all change requests is available (the document should describe that each change is authorized, tested, approved and that a global monitoring of changes is in place)

Step 2

B-The list of appropriate approvers is documented in a list which is regularly reviewed and approved

Step 3

C-special form, electronic workflow or tool (i.e. documentation) is available to address the change requests and keep track of all events

Step 4

D-All changes are tested prior to being moved to production

Step 5

E-All changes must be authorized by the appropriate person before being applied to production

Step 6

M-Emergency application program changes are handled like standard changes.

Step 7

M-SAP system openings are appropriately justified and documented.
